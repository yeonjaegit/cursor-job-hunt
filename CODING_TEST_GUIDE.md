## 🎯 핵심 원칙

### "2시간 최선을 다했으면 성공"

- ✅ 2문제 완료 → 대성공!
- ✅ 1문제 완료 + 개념 복습 → 성공!
- ✅ 1문제 도전 + 기초 문법 공부 → 성공!
- ✅ 0문제지만 2시간 집중 학습 → 성공!

## 📚 문제를 못 풀었을 때 학습 프로세스

### Step 1: 30분 도전 (09:00-09:30)
```
문제 읽기 → 이해하기 → 코드 작성 시도
```

### Step 2: 막혔을 때 (09:30)
**스스로에게 질문하기:**
- 어떤 개념이 부족한가?
- 파이썬 기본 문법 문제인가?
- 알고리즘 개념 문제인가?

### Step 3: 기초부터 공부 (09:30-10:30)

#### Case A: 파이썬 기초 부족
```python
# 리스트 기본
numbers = [1, 2, 3, 4, 5]
print(numbers[0])      # 인덱싱
print(numbers[1:3])    # 슬라이싱
numbers.append(6)      # 추가
numbers.remove(2)      # 제거

# 딕셔너리 기본
user = {"name": "연재", "age": 25}
print(user["name"])    # 값 접근
user["job"] = "개발자"  # 추가

# 반복문 기본
for num in numbers:
    print(num)
```

**Cursor에게 물어볼 것:**
```
"파이썬 리스트 기본 사용법 정리해줘"
"딕셔너리 메서드 .get(), .items(), .keys() 차이 설명해줘"
"리스트 컴프리헨션이 뭐야? 예시 보여줘"
```

#### Case B: 알고리즘 개념 부족
```
문제가 "해시"를 사용한다면
→ "해시가 뭐지?" 공부
→ "파이썬에서 해시는 딕셔너리"
→ 딕셔너리 사용법 복습
→ 비슷한 쉬운 문제 1개 풀기
```

**Cursor에게 물어볼 것:**
```
"해시 자료구조가 뭐야? 초보자도 이해할 수 있게 설명해줘"
"이 문제를 풀려면 어떤 알고리즘 개념이 필요해?"
"해시를 사용하는 프로그래머스 Level 1 쉬운 문제 추천해줘"
```

## 📈 주차별 목표

### Week 1-2: 기초 다지기
- **목표**: Level 1 쉬운 문제 10-15개
- **학습**: 파이썬 기본 문법 완벽히 익히기
- **유형**: 배열, 문자열, 반복문

### Week 3-4: 자료구조 익히기
- **목표**: Level 1 완료, Level 2 도전
- **학습**: 해시, 스택, 큐, 정렬
- **유형**: 완전탐색, 간단한 알고리즘

### Week 5+: 심화 학습
- **목표**: Level 2 집중, 틀린 문제 복습
- **학습**: DFS/BFS, 동적계획법
- **유형**: 복합 알고리즘

## 📊 일일 기록 예시

### 예시 1: 2문제 완료
```
✅ 2시간 최선을 다했는가? (완료 문제 수: 2개)
- 문제 1: 두 개 뽑아서 더하기 (20분)
- 문제 2: 완주하지 못한 선수 (30분)
- 복습: 해시 개념 정리 (20분)
```

### 예시 2: 1문제 + 학습
```
✅ 2시간 최선을 다했는가? (완료 문제 수: 1개)
- 문제 1: K번째수 (40분)
- 문제 2 도전: 실패 (30분)
- 학습: 정렬 알고리즘 개념 공부 (50분)
```

### 예시 3: 0문제지만 성공
```
✅ 2시간 최선을 다했는가? (완료 문제 수: 0개)
- 문제 도전: 체육복 (60분 시도)
- 막힌 이유: 그리디 알고리즘 개념 부족
- 학습: 
  - 파이썬 리스트 메서드 복습 (30분)
  - 그리디 알고리즘 개념 정리 (30분)
- 내일 계획: 오늘 문제 다시 도전
```
